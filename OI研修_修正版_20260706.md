開発チームやQAチームが迷わず実装・テスト・多言語展開できるように、具体的なPBI、IF仕様書、エラー定義、および実稼働するソースコードを作成してください。

> **当日の進め方（研修 7/7・演習25分）**: 当日は **Step 1〜2 ＋ Step 4 の「実装計画と最小ドラフト」（静的レビューまで）** を実施します。**Step 3・Step 5、およびサーバの起動・curl での動作確認は持ち帰り自習**です（受講PCに Node.js / Python が未導入のため、当日は起動しません）。

> **生成先**: 成果物はすべて、事前準備で作成した **`ai-training` リポジトリ内**に生成し、**ステップごとにコミット**してください。

## 3. 成果物と出力フォーマット

### Step 1: PBI（プロダクトバックログアイテム）の策定
`doc/pbi.md` を作成し、選択されたシステムの主要機能に応じたPBIを以下のフォーマットで記述してください。
1. PBIのタイトル
2. ユーザーストーリー（Who / What / Why）
3. 受入条件（Acceptance Criteria）
   - **§1 に受講者が書いた「受入条件の下書き」を出発点として**、【Given-When-Then 形式】に清書・補完してください。
   - 「正常系」および「バリデーションエラー等の異常系」を含めてください。
   - **あなた（AI）が補った受入条件には、行頭に【AI補完】と印を付けてください**（受講者が後から「自分が書いた条件」と「AIが補った条件」を見分けて検証するためです）。

### Step 2: IF仕様書（API定義書）の作成
`doc/api_specification.md` を作成し、選択されたシステムの機能に対応するAPIを定義してください。
- エンドポイント（URI、HTTPメソッド：GET/POST/PUT/DELETEなど）
- リクエスト（Headers, Query params, JSON Bodyのデータ型と必須チェック）
- レスポンス（正常系 200/201、異常系 400/404/500 のJSON構造）

### Step 3: 多言語文言定義書 ＆ エラー番号定義書の作成【自習パート】
`doc/error_and_i18n_definition.md` を作成し、以下の2つの表を記述してください。
1. **文言定義表**：システム内で使用するメッセージ（「〇〇の登録が完了しました」など）を【日本語、英語、中国語】の3言語で定義。中国語は**参考訳（未検証）**として扱い、表にその旨を注記してください。
2. **エラー番号定義表**：エラーコード（例: `E001`）、エラー内容、および各言語のメッセージの紐付け。

### Step 4: APIの実装（ソースコード）
`src/` ディレクトリ内に、実際に起動して動かせるAPIサーバーのコードを作成してください。
- 言語・フレームワーク：**§1 で受講者が指定したもの**（Node.js (Express) または Python (FastAPI)）。
- データベース：即座に動作確認ができるよう、インメモリ（配列）または SQLite を使用。
- Step 3で定義したエラー番号と多言語メッセージ（リクエストのヘッダー等で言語切り替え可能が望ましい）が組み込まれていること。
- **当日はまず「実装計画」を提示して受講者の承認を得てから、最小ドラフトを生成し、コードの静的レビュー（受入条件との突合）まで**を行ってください。**起動・動作確認は自習パート**です。

### Step 5: テストケース仕様書 ＆ 自動検証スクリプト【自習パート】
`test/` ディレクトリ内に以下を作成してください。
1. `test_cases.md`：正常系・異常系を網羅したマトリクス形式のテストケース仕様書。
2. `test_api.sh`：受講者がターミナルからAPIを叩いて、Step 3のエラーや多言語対応を即座に視覚的確認できる `curl` コマンドの実行シェルスクリプト。
