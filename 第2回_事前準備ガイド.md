## 1. 基本の考え方（重要）

- **プロジェクトの初期セットアップ（フォルダ・git・GitHub 連携・雛形作成）は、自分で手打ちせず Claude Code に依頼**します。これ自体が「エージェントに実作業を任せる」研修の予行になります。
- 仕様などの **Markdown を Obsidian から参照**できるよう、GitHub フォルダを **Obsidian が見える場所**に置きます（Mac＝iCloud Vault の中／Windows＝そのフォルダを Vault として開く）。
- **Mac は Vault を iCloud に**（iPhone/iPad 同期）。ただし **git の `.git` だけは iCloud の外**（`separate-gitdir`）＝同期競合を回避。「Markdown は同期・`.git` は安定・Obsidian 参照可」を両立。
- Claude Code が提案するコマンドは **承認前に内容を確認**してください（＝研修で学ぶ「人が検証する」の実践）。

## 【Mac版】 Vault は iCloud ／ セットアップは Claude Code に依頼

### A-1. Obsidian Vault を iCloud に作る
1. [obsidian.md](https://obsidian.md/) をインストール。
2. Vault を **iCloud 内**に作成（保存先を iCloud に）。→ iPhone/iPad 自動同期。「Mac ストレージを最適化」は**オフ**推奨。

### A-2. Vault 内に空フォルダを作り、Claude Code に依頼
1. iCloud Vault の中に空フォルダ `ai-training` を作成（Finder か Obsidian の新規フォルダ）。
2. ターミナルでそのフォルダに `cd`（**Finder からフォルダをドラッグ&ドロップ**するとパスが入る）→ `claude` を起動。
3. 次のプロンプトを貼る（GitHub のユーザー名は自分のものに）：

   > このフォルダは **iCloud 同期の Obsidian Vault の中**にあります。Git プロジェクトとして初期化したいのですが、**iCloud との競合を避けるため `.git` は iCloud の外**に置きたいです。次をお願いします。実行前に手順を一覧で見せてください。
   > 1. `~/git-repos` が無ければ作成し、**`git init --separate-git-dir ~/git-repos/ai-training.git`** でこのフォルダを初期化
   > 2. `CLAUDE.md`（プロジェクト概要の雛形）・`specs/`（仕様置き場）・`README.md`・`.gitignore` を作成
   > 3. GitHub リモート `git@github.com:<あなた>/ai-training.git` を追加（`gh` が使えればリポジトリ作成から）
   > 4. 初回コミット

4. Claude Code が提案する手順を**確認して承認** → セットアップ完了。

### A-3. Obsidian から参照
- Obsidian で **iCloud Vault を開く** → `ai-training` の Markdown（仕様など）がそのまま表示・編集できます。

> **なぜこの形か**：仕様 `.md` は iCloud で同期（Obsidian・iPhone/iPad で参照）／`.git` の重い実体は iCloud の外＝競合回避（＝講師 Mac の運用）。※ `separate-gitdir` では git 操作は Mac 本体で行い、iPhone/iPad は閲覧用。

<details><summary>フォールバック：手動でセットアップする場合（Mac）</summary>

```bash
mkdir -p ~/git-repos
cd "<あなたのiCloud Vault>/ai-training"
git init --separate-git-dir ~/git-repos/ai-training.git
git remote add origin git@github.com:<あなた>/ai-training.git
```
</details>

（Windows は iCloud と Obsidian の相性が悪いため、**演習フォルダはローカル**に置きます。）

### W-1. Obsidian をインストール

### W-2. ローカルに空フォルダを作り、Claude Code に依頼
1. ローカル（iCloud の外、例 `C:\dev\ai-training`）に空フォルダを作成。
2. そのフォルダで `claude` を起動（PowerShell で `cd C:\dev\ai-training` → `claude`）。
3. 次のプロンプトを貼る：

   > このフォルダを Git プロジェクトとして初期化してください。実行前に手順を見せてください。
   > 1. `git init`
   > 2. `CLAUDE.md`・`specs/`・`README.md`・`.gitignore` を作成
   > 3. GitHub リモート `https://github.com/<あなた>/ai-training.git` を追加（`gh` が使えればリポジトリ作成から）
   > 4. 初回コミット

4. 提案を確認して承認 → 完了。

### W-3. Obsidian から参照
- Obsidian で「**フォルダを Vault として開く**」→ `C:\dev\ai-training`。→ Markdown が表示されます。

## 付録：なぜ Mac は「Vault は iCloud／`.git` は iCloud 外」なのか
- Vault を **iCloud** に置くと iPhone/iPad と自動同期でき、仕様メモをどこでも参照・追記できます（Mac の正しい運用）。
- 一方 `.git` は大量の小ファイルを頻繁に書き換えるため iCloud と競合しやすい → **separate-gitdir で `.git` だけ iCloud の外**に。これで「Markdown は同期・git は安定・Obsidian 参照可」を同時に満たします。Claude Code への依頼プロンプトに、この `--separate-git-dir` 指定を含めています。
- Windows は iCloud×Obsidian が不安定なため、演習フォルダはローカル＋GitHub で運用します。
